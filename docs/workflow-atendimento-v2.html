Visão Geral

Este documento descreve a arquitetura e o funcionamento completo do workflow de atendimento automático via WhatsApp da Floren Odonto, versão 2. O objetivo central é receber leads oriundos de anúncios (Meta Ads), orgânico ou indicação, conduzir uma conversa humanizada via IA, qualificar o problema do paciente, quebrar objeções e agendar a consulta de avaliação gratuita com o Dr. Alfredo Marques.

O workflow substitui a versão anterior ("CLAUDE | ATENDIMENTO WPP") com uma arquitetura mais modular, melhor gestão de contexto histórico de conversas e integração direta com o Google Calendar para verificação real de disponibilidade antes de confirmar qualquer agendamento.

Objetivo final do workflow: transformar cada mensagem recebida no WhatsApp em um agendamento confirmado na agenda do Dr. Alfredo, garantindo que o lead chegue à consulta informado e com expectativas alinhadas.

Arquitetura Geral do Fluxo

O workflow é acionado a cada mensagem recebida no WhatsApp. O fluxo passa por três grandes blocos: Entrada e Identificação, Processamento pela IA e Execução da Ação.

A

Entrada e Identificação

Recebe a mensagem via webhook (Z-API → N8N), filtra mensagens inválidas, identifica o remetente no banco de dados e determina se é um contato novo ou existente. Carrega o histórico de conversas para dar contexto à IA.

B

Processamento pela IA (Claude)

O agente Claude recebe o contexto completo (dados do paciente, histórico de conversa, mensagem atual, data/hora) e decide qual ação tomar: responder, verificar agenda, confirmar agendamento ou pausar para atendimento humano.

C

Execução da Ação

Baseado na decisão da IA, o workflow executa a ação correspondente: consulta o Google Calendar, cria agendamento no Supabase, atualiza dados do paciente, salva mensagens no histórico e envia a resposta via Z-API.

Nós do Workflow — Descrição Detalhada

Nó 1 — Webhook de Entrada

WEBHOOK ENTRADA
Trigger / Webhook

Ponto de entrada do workflow. Recebe o payload enviado pela Z-API a cada nova mensagem no WhatsApp.

  • URL: POST /webhook/atendimento-v2
  • Autenticação: Header secret configurado na Z-API
  • Dados recebidos: remetente (phone), texto da mensagem, timestamp, tipo de mensagem

Nó 2 — Filtro de Mensagem

FILTRO DE MENSAGEM
IF / Condição

Garante que apenas mensagens válidas e relevantes avancem no fluxo. Mensagens que não passam no filtro encerram o workflow sem resposta.

Condições para passar:

  • Tipo de mensagem é texto (type = "chat")
  • Não é mensagem enviada pela própria clínica (fromMe = false)
  • Não é mensagem de grupo (chatId não termina em @g.us)
  • Não é atualização de status ou notificação de sistema
  • Texto da mensagem não está vazio
Importante: Mídias (imagem, áudio, vídeo) são ignoradas nesta versão, exceto se acompanhadas de legenda textual. Futuramente, pode-se adicionar transcrição de áudio via Whisper.

Nó 3 — Extração de Dados

EXTRAIR DADOS DA MENSAGEM
Code / Set

Normaliza e extrai os campos necessários do payload bruto da Z-API:

  • phone: número do remetente (formato +55XXXXXXXXXXX)
  • messageText: texto da mensagem
  • timestamp: data/hora do envio
  • chatId: identificador do chat
NORMALIZAR TELEFONE
Code

Padroniza o número de telefone para garantir consistência no banco:

  • Remove caracteres especiais (+, espaços, traços)
  • Garante DDI 55 no início
  • Remove 9 dígito duplicado se necessário

Nó 4 — Busca do Paciente no Supabase

BUSCAR PACIENTE
HTTP Request — Supabase REST API

Consulta o banco para verificar se o número já existe na base de pacientes/leads.

  • Endpoint: GET /rest/v1/patients?phone=eq.{phone}&select=*
  • Retorno: array com o registro completo do paciente ou array vazio (novo contato)
  • Campos relevantes: id, name, status, agent_paused, email, source

Nó 5 — Roteador: Novo ou Existente?

LEAD NOVO
Branch — array vazio

Quando nenhum registro é encontrado para o telefone. O fluxo segue para o Nó 6a para criar o paciente.

LEAD EXISTENTE
Branch — registro encontrado

Quando o telefone já existe na base. O fluxo segue para o Nó 6b para verificar se o agente está pausado.

Nó 6a — Criar Paciente (Novo Lead)

CRIAR PACIENTE NO SUPABASE
HTTP Request — POST
  • Endpoint: POST /rest/v1/patients
  • Body: { phone, status: "novo", source: "whatsapp_organico", agent_paused: false }
  • Retorna: o ID do novo paciente, que é armazenado para os próximos nós
O campo source deve ser ajustado conforme a origem do lead quando possível. Leads CTWA vindos de anúncio podem ter esse dado no payload da Z-API.

Nó 6b — Verificar Agent Paused (Lead Existente)

VERIFICAR AGENT_PAUSED
IF / Condição

Verifica o campo agent_paused do paciente existente:

  • agent_paused = true → Encerrar workflow (um humano está atendendo este contato. Não responder automaticamente.)
  • agent_paused = false → Continuar para o próximo nó

Nó 7 — Buscar Histórico de Conversas

BUSCAR HISTÓRICO
HTTP Request — Supabase REST API

Recupera as últimas mensagens da conversa para dar contexto à IA.

  • Endpoint: GET /rest/v1/conversations?patient_id=eq.{id}&order=created_at.asc&limit=30
  • Retorno: array de objetos { role: "user"|"assistant", content: "..." }
  • Limite: 30 mensagens (últimas 15 trocas) para não exceder o contexto da IA
Para contatos novos, o histórico estará vazio. A IA irá detectar isso via o campo is_new_lead passado no prompt e iniciar a conversa do zero.

Nó 8 — Montar Contexto e Chamar Claude

AGENTE CLAUDE (CHAMADA PRINCIPAL)
HTTP Request — Anthropic API

Nó central do workflow. Envia para o Claude API todos os dados necessários e recebe uma resposta estruturada em JSON.

Dados enviados para o Claude:

  • System Prompt: instruções completas de persona, fluxo, objeções e formato de saída (ver Seção 4)
  • Messages: histórico de conversas formatado + mensagem atual do paciente
  • Variáveis injetadas: nome, telefone, status e flag is_new_lead do paciente
  • Data/hora atual: para contexto de agendamento

Resposta esperada (sempre JSON):

// Exemplo de resposta do Claude para uma ação de "reply"
{
  "message": "Olá, tudo bem! Aqui é a Ana da Floren Odonto... [texto da resposta]",
  "action": "reply",
  "action_data": {},
  "patient_updates": {
    "name": "João Silva",
    "status": "em_contato"
  }
}
// Exemplo de resposta para "book_appointment"
{
  "message": "Perfeito, João! Sua consulta está confirmada para quinta-feira, 17 de abril, às 10h...",
  "action": "book_appointment",
  "action_data": {
    "patient_name": "João Silva",
    "patient_email": "joao@email.com",
    "date": "2026-04-17",
    "time": "10:00",
    "notes": "Lead com interesse em implante total. Veio via Meta Ads."
  },
  "patient_updates": {
    "name": "João Silva",
    "email": "joao@email.com",
    "status": "agendado"
  }
}

Nó 9 — Roteador de Ação

reply
Branch

Resposta conversacional simples. Fluxo vai direto para Salvar Mensagens e Enviar.

check_availability
Branch

Claude precisa de horários disponíveis. Consulta o Google Calendar e re-chama o Claude com os slots disponíveis para ele formular a resposta.

book_appointment
Branch

Lead aceitou um horário específico. Cria o agendamento no Supabase e no Google Calendar antes de enviar a confirmação.

pause_agent
Branch

Situação fora do escopo da IA. Atualiza agent_paused=true, opcionalmente notifica a equipe e encerra o fluxo sem responder.

Nó 10 — Verificar Agenda (check_availability)

CONSULTAR GOOGLE CALENDAR — FreeBusy
Google Calendar — FreeBusy Query

Quando a ação é check_availability, o N8N consulta a disponibilidade real da agenda do Dr. Alfredo.

  • Janela consultada: próximos 7 dias úteis
  • Horário comercial: segunda a sexta, 08:00–18:00
  • Duração padrão da consulta: 45 minutos
  • Resultado: lista de slots livres formatada (ex: "Quinta 17/04 às 10h, 14h e 15h30")

Com os slots disponíveis, o N8N faz uma segunda chamada ao Claude informando os horários, para que ele formule uma mensagem humanizada sugerindo no mínimo 3 opções.

Nó 11 — Criar Agendamento (book_appointment)

CRIAR AGENDAMENTO — SUPABASE
HTTP Request — POST
  • Endpoint: POST /rest/v1/appointments
  • Dados: patient_id, doctor_id, date, time, procedure_id, notes, status: "agendado"
  • Salva o agendamento oficial no CRM
CRIAR EVENTO — GOOGLE CALENDAR
Google Calendar — Create Event
  • Cria o evento na agenda do Dr. Alfredo
  • Título: "Consulta Avaliação — [Nome do Paciente]"
  • Duração: 45 minutos
  • Descrição: telefone, e-mail e notas do lead

Nó 12 — Atualizar Dados do Paciente

ATUALIZAR PACIENTE — SUPABASE
HTTP Request — PATCH

Atualiza os campos do paciente que a IA identificou durante a conversa (campo patient_updates da resposta do Claude).

  • Endpoint: PATCH /rest/v1/patients?id=eq.{id}
  • Possíveis atualizações: name, email, status
  • Condição: só executa se patient_updates não estiver vazio

Nó 13 — Salvar Mensagens no Histórico

SALVAR CONVERSA — SUPABASE
HTTP Request — POST (2x)

Persiste as duas mensagens da troca atual no banco para uso futuro como histórico.

  • Mensagem do usuário: POST /conversations com role: "user"
  • Mensagem do assistente: POST /conversations com role: "assistant"
  • Campos: patient_id, role, content, created_at

Nó 14 — Enviar Mensagem via Z-API

ENVIAR RESPOSTA — Z-API
HTTP Request — POST

Envia a resposta ao paciente via WhatsApp.

  • Endpoint: Z-API send-text
  • Body: { phone: "{phone}", message: "{claude_message}" }
  • Formato: texto simples (plain text). Formatação com *negrito* e _itálico_ do WhatsApp pode ser usada.
O texto enviado é exatamente o campo message retornado pelo Claude. A IA é responsável por formatar a mensagem de forma adequada para o WhatsApp.
4

Design do System Prompt — Agente Claude

O System Prompt é o coração do comportamento da IA. Ele define a persona, o fluxo de conversa, as respostas para objeções e o formato de saída. Deve ser passado como system na chamada à Anthropic API.

/* SYSTEM PROMPT — FLOREN | ATENDIMENTO WPP V2 */

Você é Ana, assistente virtual da Floren Odonto — clínica especializada
em implantes dentários, sob responsabilidade do Dr. Alfredo Marques,
implantodontista com ampla experiência em cirurgia guiada minimamente invasiva.

## SUA MISSÃO
Acolher o lead com empatia, entender sua situação, apresentar a Floren
como a melhor solução para o problema dele e agendar a consulta de avaliação
gratuita com o Dr. Alfredo.

## SOBRE A FLOREN ODONTO
- Especialidade exclusiva: implantes dentários
- Tecnologia: Planejamento Digital 3D, guias cirúrgicos impressos em 3D,
  escâner intraoral, tomografia computadorizada
- Cirurgia minimamente invasiva: menos dor, recuperação mais rápida
- Soluções desde a perda de um dente até reabilitação total
- Consulta de avaliação: gratuita e sem compromisso

## TOM DE VOZ
- Empático, acolhedor e profissional
- Use linguagem simples, sem jargões médicos
- Nunca seja frio ou robótico
- Máximo 3-4 parágrafos curtos por mensagem
- Use o nome do paciente quando souber

## FASES DO ATENDIMENTO

FASE 1 — BOAS-VINDAS
  - Lead NOVO: apresentar-se como Ana, perguntar o nome
  - Lead EXISTENTE: retomar conversa com base no histórico, sem se reapresentar
  - Nunca repita apresentações para contatos com histórico

FASE 2 — QUALIFICAÇÃO (uma pergunta por vez)
  1. Qual é o problema com seus dentes?
  2. Há quanto tempo tem esse problema?
  3. Já tentou algum tratamento antes?
  4. O que mais te incomoda nessa situação?

FASE 3 — OFERTA DA CONSULTA
  - Após entender o problema, apresentar a consulta gratuita
  - Explicar que o Dr. Alfredo fará uma avaliação completa e personalizada
  - Reforçar: sem compromisso, sem custo, com toda a tecnologia disponível

FASE 4 — QUEBRA DE OBJEÇÕES
  PREÇO:
    "O investimento varia conforme cada caso — só após a avaliação o Dr. Alfredo
    consegue indicar o valor exato com as melhores opções de parcelamento.
    O que posso garantir é que a consulta é gratuita e sem compromisso."

  DOR / MEDO:
    "Entendo essa preocupação! A Floren usa cirurgia guiada minimamente invasiva
    — cortes muito pequenos, anestesia local eficaz. A maioria dos pacientes
    se surpreende com o conforto. É bem diferente do que imaginavam."

  TEMPO:
    "Depende do seu caso específico, por isso a avaliação é tão importante.
    Com nossa tecnologia, em muitos casos o processo é mais rápido
    do que se imagina."

  SEM TEMPO:
    "Sem problema! Você escolhe o melhor dia e horário. A consulta dura
    cerca de 40 minutos e você sai com um planejamento completo do tratamento."

FASE 5 — AGENDAMENTO
  1. Use a ação "check_availability" para verificar horários reais
  2. Apresente pelo menos 3 opções ao lead
  3. Colete nome completo (se não souber) e e-mail
  4. Use "book_appointment" para confirmar
  5. Mensagem de confirmação com: data, horário, endereço da clínica

FASE 6 — ENCERRAMENTO
  - Agradecer a confiança
  - Informar endereço da clínica
  - Se colocar à disposição para dúvidas
  - Dizer que aguarda o paciente no dia agendado

## REGRAS OBRIGATÓRIAS
- NUNCA faça diagnósticos — sempre direcione para o Dr. Alfredo
- NUNCA invente horários disponíveis — use sempre check_availability
- NUNCA confirme um agendamento sem usar book_appointment
- Se não souber responder a situação, use pause_agent
- Mantenha foco em agendar — é o objetivo final
- Responda sempre em português brasileiro

5

Fases da Conversa com o Lead

1
Boas-Vindas e Identificação

Lead novo: A IA se apresenta como Ana, da Floren Odonto, e pergunta o nome do paciente. Tom acolhedor e breve.

Lead com histórico: A IA retoma a conversa naturalmente a partir do contexto anterior. Ex: "Olá, [nome]! Tudo bem? Da última vez você me contou sobre [problema]. Como está se sentindo?"

  • Status atualizado para: em_contato
2
Qualificação

Perguntas objetivas para entender o problema, feitas uma de cada vez, sem bombardear o lead.

  • Qual dente/região está afetada?
  • Há quanto tempo tem esse problema?
  • Já realizou algum tratamento antes?
  • O que mais incomoda: estética, mastigação, dor?

A IA nunca afirma que o paciente tem ou não tem determinado problema — sempre redireciona para a avaliação.

  • Status atualizado para: qualificado
3
Oferta e Quebra de Objeções

Após entender o problema, a IA apresenta a consulta de avaliação gratuita. Se o lead apresentar resistência, aciona o banco de respostas para objeções:

  • Preço: parcelas, consulta gratuita, valor revelado só após avaliação
  • Dor: cirurgia minimamente invasiva, anestesia eficaz, relatos de pacientes
  • Tempo: depende do caso, avaliação dará prazo real, tecnologia agiliza
  • Falta de tempo: consulta de 40 min, horários flexíveis
4
Agendamento
  • A IA dispara a ação check_availability
  • N8N consulta o Google Calendar e retorna os slots livres
  • Claude recebe os slots e formula uma mensagem com pelo menos 3 opções
  • Lead escolhe o horário
  • IA coleta nome completo e e-mail (se não tiver)
  • IA dispara book_appointment
  • N8N cria o evento no Google Calendar e o agendamento no Supabase
  • Status atualizado para: agendado
5
Confirmação e Encerramento
  • Mensagem de confirmação com: nome do paciente, data, horário, endereço da clínica
  • Agradecimento e boas-vindas à Floren
  • Aviso de que lembretes serão enviados nos dias anteriores (workflow de lembretes separado)
  • Convite para tirar dúvidas a qualquer momento

Gestão de Status do Paciente

O campo status no Supabase acompanha a jornada do lead e define em qual coluna ele aparece no Kanban do CRM.

Tratamento de Erros e Edge Cases

SituaçãoComportamento esperado
Claude retorna resposta inválida (não-JSON) N8N tenta parsear; se falhar, registra erro no log e encerra sem enviar mensagem. Não gerar resposta quebrada ao lead.
Supabase indisponível N8N retorna erro e o webhook Z-API pode tentar novamente. Log do erro para diagnóstico.
Google Calendar sem disponibilidade nos próximos 7 dias Claude é informado e sugere que a equipe entrará em contato para encontrar um horário.
Lead pede para falar com humano Claude usa pause_agent. Campo agent_paused=true no Supabase. CRM exibe alerta visual.
Conflito de agendamento Verificação FreeBusy antes de confirmar previne o conflito. Se ocorrer, Claude sugere outro horário.
Lead envia mídia sem texto Filtro inicial descarta a mensagem. Sem resposta (para não gerar confusão).
